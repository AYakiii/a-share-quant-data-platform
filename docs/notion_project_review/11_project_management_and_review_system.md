# 11 Project Management and Review System

## 长期维护机制
- 每个 Phase 完成后，必须更新 `12_project_tracking_log_template.md` 实例页。
- design decision 需记录“动机-方案-取舍-验证结果-回滚条件”。

## MVP vs 长期架构
- 在每条变更中标记：`MVP temporary` 或 `long-term architecture candidate`。

## technical debt 追踪
- 按模块维护 debt 清单：触发条件、影响范围、偿还优先级。

## 防止文档过时
- 将文档更新作为 PR checklist 条目。
- 每月做一次“代码-文档一致性抽检”。
